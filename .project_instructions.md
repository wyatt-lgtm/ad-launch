# Ad Launch - Project Instructions

## Overview
Ad Launch is a lead-gen platform that takes a business URL, performs deep research via Tombstone OS (FastAPI backend), and generates visual Facebook ads, SEO audits, Google Ads copy, website concepts, budget recommendations, and 90-day posting plans.

## Architecture
- **Frontend:** Next.js 14 (App Router) with Tailwind CSS, deployed at ad-launch-1nfyr8.abacusai.app + connect.launchmarketing.com
- **Backend Pipeline:** Tombstone OS (FastAPI on Render) ‚Äî multi-agent system for research + creative generation
- **Database:** PostgreSQL (Prisma ORM)
- **Auth:** NextAuth with credentials (business email only)
- **CRM:** GoHighLevel (GHL) for contact management + confirmation emails
- **Image Storage:** S3 (AWS) for gpt-image-1 generated ad images; R2 (Cloudflare) for legacy Tombstone artifacts
- **GitHub Repos** (May 2026 separation):
  - `wyatt-lgtm/ad-launch` (branch: `main`) ‚Äî Ad Launch frontend (this project)
  - `wyatt-lgtm/tombstone` (branch: `main`) ‚Äî Tombstone OS backend (separate Render deploy)
  - `.env` stripped from git history in both repos

### Business-Level Content Source Mode (May 2026)
- `Business.contentSourceMode` field with 3 modes: `local_only`, `local_plus_interests`, `interests_only`
- Default: `local_plus_interests` for all existing businesses
- `BusinessFeedPreference` model: per-business industry/category toggles (`@@unique([businessId, industry])`)
- API: `GET/PATCH /api/businesses/[id]/content-settings` ‚Äî returns/updates mode + selected categories + industry metadata
- Clark Kent (`/api/rss/clark-kent`) accepts `businessId` body param, loads mode from business, controls:

  - `local_plus_interests`: local RSS + interest feeds + events
  - `interests_only`: interest feeds + events (no ZIP required)
- Interest brief: `lib/rss/interest-feed-brief.ts` ‚Äî `generateInterestFeedBrief(businessId)` queries national/active feeds by enabled BusinessFeedPreference industries ‚Üí returns structured brief
- `formatInterestBriefForCommand()` ‚Äî text block for Tombstone command embedding
- `createSocialMissions()` accepts `contentSourceMode` option, adapts lane instructions per mode
- Content Feeds UI (`/dashboard/feeds`): mode selector (3 cards) above interest category grid, saves to business-level API, hides grid in `local_only`, passes businessId to Clark Kent + social/generate
- Old `UserFeedPreference` model + `/api/user/feed-preferences` kept for backward compat; new flow uses `BusinessFeedPreference` via content-settings API
