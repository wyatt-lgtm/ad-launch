# Ad Launch - Project Instructions

## Media War Room + QA System (May 2026)

### Pre-Generation War Room (`lib/media-war-room.ts`)
- `reviewMediaConceptBeforeGeneration(input)` — Reviews creative concept before expensive media generation
- 11 hard-fail rules: generic/boring, visual-topic mismatch, CTA mismatch, instant-read, forced connection, vague prompt, leaked internals, restricted claims, stock-likely, text overflow, weak hook
- Returns: `{ decision: 'approved'|'revise'|'reject', score, failReasons, revisedGenerationPrompt, ... }`
- On 'revise': provides improved prompt/headline/CTA for generation to use
- On 'reject': blocks generation entirely
- On LLM failure: pass-through (score=-1) so pipeline is never blocked

### Post-Generation QA (`lib/media-qa.ts`)
- `reviewRenderedMediaBeforeReady(input)` — Reviews rendered image after generation
- 10 hard-fail rules: blank space, CTA mismatch, subject relevance, text integrity, business connection, stock generic, composition, aspect ratio, text readability, brand alignment
- Replaces/supersedes the older `lib/post-qa.ts` (still exists for reference)

### Integration Points
- **`completion-check/route.ts`** (Tombstone social posts): War Room concept review → Final QA → then promote to `ready`
- **`generate-ad-image.ts`** (GPT-5.1 ad images): War Room pre-gen → image generation → Post-gen QA
- **`edit-ad/route.ts`** (user image edits): War Room pre-gen → image generation → Post-gen QA
- All paths: on reject → no credit charge, no email, reasons stored in DB

### Schema Fields (PostPackage)
- `warRoomStatus` — 'approved' | 'revise' | 'reject' (null if not run)
- `warRoomScore` — 0-100 concept quality score
- `warRoomRejectReason` — pipe-separated failure reasons
- `qaScore` — 0-100 rendered media quality score
- `qaRejectReason` — pipe-separated failure reasons

### Searchable Log Events
- `MEDIA_WAR_ROOM_REVIEW_STARTED/PASSED/REVISED/REJECTED`
- `MEDIA_FINAL_QA_STARTED/PASSED/REJECTED/ERROR_PASS_THROUGH`

### Notes
- Both modules use `gpt-5.4-mini` via Abacus AI LLM API
- For Tombstone workflows: War Room runs post-completion (can't intercept mid-pipeline from Ad Launch) but still catches concept-level issues
- For direct GPT-5.1 generation: War Room truly prevents wasteful generation
