## Business Profile Reuse (Social Posts)

### What it does
When a social post workflow runs, Jim Bridger's recon output (business_context) is saved to `Business.savedAnalysis`. On subsequent runs, the saved profile is injected into the Tombstone command as a `--- SAVED BUSINESS PROFILE ---` block, and Jim Bridger skips full website scraping + LLM analysis.

### Key files
- `lib/business-profile.ts` — save/load/format utilities, staleness check (30 days)
- `lib/tombstone.ts` — `getBusinessProfileBlock()` helper, injected in all mission creators
- `app/api/scout/completion-check/route.ts` — saves profile when workflow completes
- `/home/ubuntu/tombstone/agents/jim_bridger.py` — `_extract_saved_business_profile()` parser

### Schema additions
- `Business.savedAnalysis` (Json?) — cached recon data
- `Business.analysisRefreshedAt` (DateTime?) — when last refreshed

### Log prefixes
- `SOCIAL_BUSINESS_CONTEXT_LOADED` — profile found and loaded
- `SOCIAL_BUSINESS_CONTEXT_MISSING_ANALYSIS` — no saved profile, will run fresh
- `SOCIAL_BUSINESS_CONTEXT_REFRESH_STARTED` — profile stale (>30d), refreshing
- `SOCIAL_BUSINESS_CONTEXT_REUSED` — Jim Bridger skipped scraping

### UI label
- "Business Analysis" renamed to "Business Context" in DEPT_LABELS
- Progress message shows "Loaded your saved business profile." when reusing

## Portrait/Mobile-First Social Post Images (May 2026)

### What it does
All RSS/Content Feed/Scout Stories social post images now default to **4:5 portrait** (1080×1350) instead of landscape/square. This optimizes for mobile feeds where most users view content.

### Changes by layer
| Layer | File | Change |
|-------|------|--------|
| Command injection | `lib/tombstone.ts` | `buildStoryCommand` + `createScoutStoryMission` include portrait instruction |
| Creative direction | `don_draper.py` | IMAGE ORIENTATION RULE in LLM prompt: default 4:5, 9:16 for Stories only, 16:9 only if explicitly needed |
| Render production | `andy_warhol.py` | Default `aspect_ratio` changed from `1:1` → `4:5`; full-comp prompt says "mobile-first portrait" |
| War Room (Tombstone) | `bat_masterson.py` | New rule 12: `LANDSCAPE_SOCIAL_FEED` — rejects landscape for social feed; `_apply_revisions` forces 16:9 → 4:5 |
| War Room (Ad Launch) | `lib/media-war-room.ts` | New rule 12: `LANDSCAPE_SOCIAL_FEED` |
| QA (Ad Launch) | `lib/media-qa.ts` | Rule 8 renamed `ASPECT_RATIO_MISMATCH` with explicit landscape-for-social rejection |
| Progress labels | `app/api/social/progress/route.ts` | "Creating portrait social image", "Preparing feed-ready image", "Generating mobile-ready creative" |

### Backward compatibility
- Existing older landscape posts are NOT modified
- Only applies to new generations going forward
- User can still explicitly request landscape if needed
