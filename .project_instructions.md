### Scout Geo Fallback Cascade (June 2026)
- **Problem**: Local news came up blank for businesses outside the seeded TX geo area (e.g., Buffalo NY). Only ZIP-radius search via FeedGeo existed; no geographic widening when results were thin.
- **Root cause**: `FeedGeo` table had 17,416 rows all concentrated on TX ZIPs. Zero entries for NY, and no `pilotState='NY'` feeds.
- **Fix**: `generateContentBriefWithFallback()` in `lib/rss/trade-area-feed.ts` implements a 5-level cascade:
  1. **ZIP radius** (initial lookback days)
  2. **City** (2× lookback via `getTradeAreaItems({ cities })`)
  3. **County** (full LOOKBACK_DAYS via `getTradeAreaItems({ counties })`)
  4. **State** (full LOOKBACK_DAYS via `getTradeAreaItems({ states })`)
  5. **National-only** (queries `geoScope='national'` feeds directly)
- **Auto-discovery**: When results < `SCOUT_MIN_LOCAL_ITEMS` after all levels, `triggerAsyncDiscovery()` probes candidate news sites (cityname+news/patch/press/times/etc.) and saves discovered feeds with `status: 'pending'` for manual activation.
- **Env vars**: `SCOUT_MIN_LOCAL_ITEMS=5`, `SCOUT_MAX_LOCAL_ITEMS=20`, `SCOUT_LOCAL_NEWS_LOOKBACK_DAYS=14`, `SCOUT_FEED_DISCOVERY_ENABLED=true`
- **Diagnostics**: `BriefDiagnostics` interface tracks fallbackLevel, levelsAttempted, discoveryTriggered, queryTimeMs. Exposed in Clark Kent's `meta.rssDiagnostics`.
- **Re-ranking**: Composite score includes geo specificity bonus (zip:10, city:7, county:4, state:2, national:1), feed diversity cap (max 3 per feed), title dedup.
- **Clark Kent route**: Now calls `generateContentBriefWithFallback()` with city/state context. Scout summary shows `[geo fallback: county]` notes and per-headline `[locality_level]` tags.
- **Backwards compatible**: `generateContentBrief()` still works (delegates to fallback version). `ContentBrief` interface has optional `diagnostics` and `localityLevel` fields.
- **Key helpers used**: `getZipDetails()` resolves ZIP → city/county/state. `getTradeAreaItems()` already supported `cities`, `counties`, `states` params.

#### Geo Linker & Feed Discovery Pipeline (June 2026)
- **`lib/rss/geo-linker.ts`** — comprehensive feed discovery + validation + FeedGeo linking:
  - `validateFeed()`: Fetches feed, parses XML, verifies item count/recency. Invalid feeds rejected before saving.
  - `createFeedGeoLinks()`: Creates `FeedGeo` rows linking a feed to all ZIPs in a city/county/state scope. Uses existing `getZipsByCity`/`getZipsByCounty`/`getZipsByState`.
  - `inferGeoFromFeed()`: Deterministic geo inference from known-domain map (`CITY_DOMAIN_MAP`), Patch.com URL pattern, state names in title, `pilotState` field.
  - `discoverValidateAndLink()`: Full pipeline: build candidate site list → discover feeds via HTML+path probing → validate each → save `RssFeed` → create `FeedGeo` links. Used by both async scout discovery and manual admin trigger.
  - `backfillFeedGeo()`: For existing feeds without `FeedGeo` rows, infers geo and creates links. Reports uncertain feeds instead of guessing.

- **De-duplication**: Items now de-duped by both ID and normalized title (lowercase, stripped punctuation, first 80 chars). Prevents duplicate headlines from overlapping feeds.

### Visual Residue / Cross-Business Image Contamination (June 2026)
- **Problem**: Generated website concepts and social posts sometimes contain imagery from unrelated businesses (e.g., wifi/router/ISP imagery appearing on a cupcake shop's website). This is cross-business context contamination in the image generation pipeline.
- **Root cause**: Tombstone backend — Don Draper's image direction and Andy Warhol's render prompts may pick up cached context, memory, or asset references from prior workflows for different businesses. The identity lock block prevents TEXT contamination but doesn't scope IMAGE generation prompts.
- **Where to fix (Tombstone backend)**:
  1. Don Draper (`agents/don_draper.py`): Image direction schema must include `business_id` scoping. The `must_show` and `must_avoid` fields should be validated against the current business's asset library only.
  2. Andy Warhol (`agents/andy_warhol.py`): Render prompt should include explicit business-scoped image context (business name, industry, website) to prevent cross-contamination.
  3. Memory service: Ensure `business_memory` lookups are scoped by `business_id` and never return assets/context from other businesses.
  4. QA gate (Peter Drucker): Add a visual coherence check — flag renders whose content doesn't match the advertiser's industry/domain.
- **Frontend side**: No fix needed. Asset retrieval (`/api/assets/approved`) is already properly scoped by `businessId`/`tombstoneBusinessId`. The contamination happens during prompt construction on the Tombstone side.
- **Mitigation added**: Enhanced logging in `getWorkflowResults()` and `mission-status` to track lane→workflow→ad mappings for post-incident diagnostics.
