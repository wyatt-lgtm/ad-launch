# Ad Launch - Project Instructions

## Overview
Ad Launch is a lead-gen platform that takes a business URL, performs deep research via Tombstone OS (FastAPI backend), and generates visual Facebook ads, SEO audits, Google Ads copy, website concepts, budget recommendations, and 90-day posting plans.

## Architecture
- **Frontend:** Next.js 14 (App Router) with Tailwind CSS, deployed at ad-launch-1nfyr8.abacusai.app
- **Backend Pipeline:** Tombstone OS (FastAPI on Render) — multi-agent system for research + creative generation
- **Database:** PostgreSQL (Prisma ORM)
- **Auth:** NextAuth with credentials (business email only)
- **CRM:** GoHighLevel (GHL) for contact management + confirmation emails
- **Image Storage:** S3 (AWS) for GPT-5.1 generated ad images; R2 (Cloudflare) for legacy Tombstone artifacts

## Ad Generation Pipeline (GPT-5.1 Upgrade - April 2026)
- **Previous:** Andy Warhol (FAL Flux) → Claude Hopkins (text overlay) → basic stock photo with text
- **Current:** After Tombstone completes research + creative, `mission-status/route.ts` calls `generateAllAdImages()` which uses **GPT-5.1 "Designer Brief"** prompt pattern to generate professional multi-section Facebook ad compositions
- **Prompt Strategy:** Agency brief style ("You are a senior graphic designer...") with structured sections: CLIENT, CREATIVE BRIEF, DESIGN DIRECTION
- **Fallback:** If GPT-5.1 generation fails, falls back to original Tombstone FAL images
- **Storage:** GPT-5.1 images are uploaded to S3 as public PNGs, URL stored in Ad.imageUrl
- **Edit Ads:** `/api/edit-ad` also uses GPT-5.1 instead of gpt_image15

## Key Models Used
- `gpt-5.1` — Ad image generation (Designer Brief pattern), ad editing

### Phase 10: Clark Kent Scout → Tombstone Social Pipeline (complete)
- **Clark Kent = Scout ONLY**: `POST /api/rss/clark-kent` gathers local intelligence but does NOT create posts or call LLMs

  - Gathers upcoming events via `lib/social/upcoming-events.ts`
  - Extracts business context from Analysis (industry, core offer, value props)
  - Outputs `ScoutBrief` with human-readable `scoutSummary` for Tombstone agents
- **Tombstone Creative Workflow**: Scout brief is sent as a Tombstone command via `createSocialMissions()`
  - Zig Ziglar receives Clark's intel + business research → creates marketing angles
  - Ogilvy → Don Draper → Andy Warhol → Claude Hopkins → produces finished posts WITH artwork
  - 3 lanes: Local News (3), Business Promo (3), Seasonal/Holiday (3) = 9 posts
- **Integration Flow** (`confirm-and-launch`):
  1. Clark Kent scouts (background, fire-and-forget)
  2. Scout brief → `createSocialMissions()` → Tombstone social workflow
  3. `socialMissionId` stored on Analysis for polling
  4. `mission-status` polls social workflow via `getSocialWorkflowResults()`

### Ad Duplication Race Condition Fix (April 2026)
- Bug: `mission-status/route.ts` used `status: { not: 'completed' }` for atomic lock, but `completing` also passed the filter
- Fix: Changed to `status: { notIn: ['completed', 'completing'] }` so only the first concurrent request wins the lock
- Result: Exactly 3 ads created per analysis (previously could create 141+ from concurrent poll requests)
